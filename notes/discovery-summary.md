# Major Discovery — 2026-06-14

**Finding:** The self-hosted compiler (`src/m1/m1c.m0`) has almost no temporal language support.

## Details

- `src/m1/lexer.m0` defines tokens for temporal keywords (`TkFreeze`, `TkNow`, `TkWwas`, `TkWhen`, `TkWill`, `TkLive`).
- `src/m1/m1c.m0` only contains the five `phase_graph_*` external declarations (lines 19–23).
- There are **zero** parser productions for temporal constructs.
- There are **zero** AST node kinds (`N_FREEZE`, `N_WAS`, `N_WILL`, `N_NOW`, `N_LIVE`) defined or used in the self-hosted compiler.
- There is **zero** lowering or codegen for `live`, `freeze`, `was`, `will`, `when/then`, or `now`.

**Contrast:** The C bootstrap (`src/codegen.c`, `src/m1/phase_graph.c`, etc.) has full support for all temporal constructs and Phase Graph integration. The golden evidence in `paper/evidence/` was generated by the C bootstrap — not by `m1c.m0`.

## Implications for v0.6

The original plan assumed temporal constructs mostly existed in `m1c.m0` and that we primarily needed to wire the Phase Graph. **Reality:** We must first port the temporal frontend (parser + AST + basic lowering) from the C bootstrap into the self-hosted compiler before meaningful Phase Graph integration or temporal semantics can happen.

This is a significantly larger body of work than originally scoped for the first 30 days.

## Recommended Revised Near-Term Focus

1. Port temporal parser productions + AST nodes into `m1c.m0`.
2. Implement basic lowering that can call the existing `phase_graph_*` externals.
3. Then do the "real" Phase Graph integration (constant folding, conflict detection).
4. Address `now` re-check, self-hosting stability, etc. after the above.
